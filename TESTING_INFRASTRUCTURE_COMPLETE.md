## How to Use This Infrastructure

### For Developers

#### Before Committing
```bash
# Run fast tests
go test -short ./...

# Check your changes
git add .
git commit -m "Your message"
# Pre-commit hooks run automatically
```

#### Before Pushing
```bash
# Run full test suite
./scripts/test_orchestrator.sh

# Check coverage
cat coverage/SUMMARY.md

# View detailed coverage
open coverage/coverage.html
```

#### Adding New Tests
```bash
# Create test file
touch pkg/mypackage/myfile_test.go

# Use table-driven tests (see guide)
# Run tests
go test -v ./pkg/mypackage/...

# Check coverage
go test -coverprofile=coverage.out ./pkg/mypackage/...
go tool cover -html=coverage.out
```

### For CI/CD

The CI/CD pipeline runs automatically on:
- Every push to main/develop/feature branches
- Every pull request
- Daily at 2 AM UTC
- Manual trigger

**Pipeline Ensures**:
- Code is formatted
- Code passes linting
- Tests pass on multiple platforms
- Coverage meets minimum threshold (85%)
- No security vulnerabilities
- Builds succeed

### For QA/Testing

```bash
# Run all tests with verbose output
./scripts/test_orchestrator.sh

# Run integration tests only
go test -tags=integration -v ./...

# Run benchmarks
./scripts/test_orchestrator.sh --run-benchmarks

# Check specific package
go test -v -cover ./pkg/probe/...
```
